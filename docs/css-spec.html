CSS 规范

概述

规范的目的在于使我们的CSS代码更加易于维护和重用，从而提升效率。

选择器命名
  • 命名除 .fn- / .ui- / .sl- 外，可自定义命名。命名以简明扼要为原则。
  • 命名包含多个单词时，使用 “-” 连接，如果 .ui-pop 。
  • 一般情况下，如果命名比较通用，比如 .current ，请限定在相应的上下文环境中。建议写成 #parent .current 。
  • 作为 JS 接口的 class 和 ID ，必须以 .js- 开头。
选择器书写

多于2个的选择器使用同样的属性时，每个各占一行。

.selector1,
.selector2,
.selector3,
...
.selectorN { property: value; }

避免使用低效的选择器

body > * {...}
div#ID {...}
div#ID > h3 {...}
#ID span.class a {...}
div:last-child {...}
注释

在文档的顶部声明组件名，组件介绍，依赖的样式，作者。

/**
 * @name: 组件名
 * @overview: 组件介绍
 * @require: 依赖的样式
 * @author: 作者
 */

按功能模块注释，在模块的结尾空一行表示结束。

/* news list */
.list li {...}
.list li a {...}

/* gallery list */
.gallery li {...}
.gallery li a {...}
